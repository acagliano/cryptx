Before you do anything with the libraries, you must download the project. You can do this by downloading the latest release from the project repository. Alternatively, you can clone the repository by running the following command in Terminal or Command Prompt:
git clone https://github.com/acagliano/cryptx.git.

Sending to your Calculator

The binaries within CryptX are distributed as a single package or as individually libraries. For the former, you will want to seek out the CryptX.8xg file in the downloaded package. If you want individual libraries, look into the HASHLIB, ENCRYPT, and ENCODEX directories for a .8xv file sharing the same name as the library. Once you have located the binaries you want to install, you will want to connect your calculator to your computer using a USB-A to mini-B cable and use your preferred connectivity software to transfer the binaries to your device.

Installing to the Toolchain (for developers)

If you are a developer who wants to use CryptX in one of your own projects, it gets a bit more complicated. In order to build as a dynamic library (which is how CryptX is designed) you will need the .lib and .h files for each library moved into the appropriate directories within the toolchain.

File ExtensionTarget Directory
.h<CEdev_root>/include
.lib<CEdev_root>/lib/libload

Once this is done, you should be able to include the library headers in your project the same way you would any other C standard library.
Ex: #include <encrypt.h>

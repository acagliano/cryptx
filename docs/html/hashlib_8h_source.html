38/*
39Cryptographic Hashes
40
41A cryptographic hash is used to validate that data is unchanged between two endpoints.
42It is similar to a checksum, but checksums can be easily fooled; cryptographic hashes
43are a lot harder to fool due to how they distribute the bits in a data stream.
44The general use of a hash is as follows: the party sending a message hashes it and
45includes that hash as part of the message. The recipient hashes the message (except the hash)
46themselves and then compares that hash to the one included in the message. If the hashes match,
47the message is complete and unaltered. If the hashes do not match, the message is incomplete
48or has been tampered with.
49*/

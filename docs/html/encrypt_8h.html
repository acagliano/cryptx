Detailed Description

Provides standard encryption for the TI-84+ CE.

Author
Anthony ACagliano Cagliano
Adam beck Beckingham
commandblockguy

Provides symmetric and asymmetric (pubkey) encryption as well as a HWRNG.

  1. Secure HWRNG
  2. AES-128, AES-192, AES-256
  3. RSA + OAEP v2.2

aes_init
aes_error_t aes_init(aes_ctx *ctx, const void *key, size_t keylen, const void *iv, uint24_t flags)
Initializes a stateful AES cipher context to be used for encryption or decryption.
Note
Do not edit a context manually. You may corrupt the cipher state.
Contexts are not bidirectional due to being stateful. If you need to process both encryption and decryption, initialize seperate contexts for encryption and decryption. Both contexts will use the same key, but different initialization vectors.
Warning
It is recommended to cycle your key after encrypting 2^64 blocks of data with the same key.
Do not manually edit the ctx.mode field of the context structure. This will break the cipher configuration. If you want to change cipher modes, do so by calling aes_init again.
AES-CBC and CTR modes ensure confidentiality but do not guard against tampering. AES-OCB/GCM are a bit computationallty-intensive for this platform, but HASHLIB provides hash and hmac functions in their stead. HMAC is generally more secure for this purpose. If you want a truly secure scheme, always append an HMAC to your message and use an application secret or unique key generated using a CSRNG to key the HMAC at both endpoints.
Returns
AES_OK if success, non-zero if failed. See aes_error_t.

General-Purpose AES Encryption.

Parameters
ctxPointer to an AES cipher context.
plaintextPointer to data to encrypt.
lenLength of data at plaintext to encrypt. This can be the output of hashlib_AESCiphertextSize().
ciphertextPointer to buffer to write encrypted data to.
Note
ciphertext should large enough to hold the encrypted message. For CBC mode, this is the smallest multiple of the blocksize that will hold the plaintext, plus 1 block if the blocksize divides the plaintext evenly. For CTR mode, this is the same size as the plaintext.
plaintext and ciphertext are aliasable.
Encrypt is streamable, such that encrypt(msg1) + encrypt(msg2) is functionally identical to encrypt(msg1+msg2) with the exception of intervening padding in CBC mode.
Once a context is used for encryption, a stateful flag is set preventing the same context from being used for decryption.
Returns
AES_OK if success, non-zero if failed. See aes_error_t.

RSA Encryption.

Performs an in-place RSA encryption of a message over a public modulus pubkey and a public exponent, 65537 OAEP encoding of the input message is performed automatically.

Parameters
msgPointer to a message to encrypt using RSA.
msglenThe length of the message msg.
ciphertextPointer a buffer to write the ciphertext to.
pubkeyPointer to a public key to use for encryption.
keylenThe length of the public key (modulus) to encrypt with.
oaep_hash_algThe numeric ID of the hashing algorithm to use within OAEP encoding. See hash_algorithms.
Note
The size of ciphertext and keylen must be equal.
The msg will be encoded using OAEP before encryption.
msg and pubkey are both treated as byte arrays.

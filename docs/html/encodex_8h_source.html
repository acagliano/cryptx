152// #################################
153// ###### *N*Bpp Byte Packing ######
154// #################################
155/*
156 * Bits-per-pixel is a a form of data compression in which only the
157 * active bits of a series of bytes are retained and the bytes are
158 * compressed such that each octet contains multiple "bytes" worth of data.
159 * For example, imagine a 4-byte long data stream with possible values
160 * 0x00 - 0x03. In this scheme, only the low 2 bits of each byte are
161 * actually used (0x00 = 0b00000000, 0x01 = 0b00000001, 0x02 = 0b00000010,
162 * 0x03 = 0b00000011). This means we are wasting 6 bits, or 75% of the
163 * data used to represent it.
164 * The correct encoding scheme to use here is 2-bpp (2 bits-per-pixel).
165 * This would express each "byte" in the stream as two bits, allowing us
166 * to compress the data, previously 4 bytes, into a single byte, losslessly.
167 */

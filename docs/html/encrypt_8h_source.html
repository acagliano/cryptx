18/*
19 Cryptographically-Secure Random Number Generator (CSRNG)
20
21 Many of the psuedorandom number generators (PRNGs) you find in computers and
22 even the one within the C toolchain for the CE are insecure for cryptographic
23 purposes. They produce statistical randomness, but the state is generally seeded
24 using a value such as rtc_Time(). If an adversary reconstructs the seed, every
25 output of the PRNG becomes computable with little effort. These types of PRNGs
26 are called deterministic algorithms--given the input, the output is predictable.
27 These PRNGs work for games and other applications where the illusion of randomness
28 is sufficient, but they are not safe for cryptography.
29
30 A secure PRNG is an random number generator that is not only statistically random, but
31 also passes the next-bit and state compromise tests. The _next-bit test_ is defined like so:
32 given the prior output of the PRNG (bits 0=>i), the next bit (i+1) of the output cannot be
33 predicted by a polynomial-time statistical test with a probability non-negligibly greater than 50%.
34 In simpler terms, a secure PRNG must be unpredictable, or "entropic".
35 The _state compromise test_ means that an adversary gaining knowledge of the initial state of
36 the PRNG does not gain any information about its output.
37
38 The SRNG previded by HASHLIB solves both tests like so:
39 (next-bit)
40 <> The SRNG's output is derived from a 119-byte entropy pool created by reading
41 data from the most entropic byte located within floating memory on the device.
42 <> The "entropic byte" is a byte containing a bit that that, out of 1024 test reads,
43 has the closest to a 50/50 split between 1's and 0's.
44 <> The byte containing that bit is read in xor mode seven times per byte to offset
45 any hardware-based correlation between subsequent reads from the entropic byte.
46 <> The entropy pool is then run through a cryptographic hash to spread that entropy
47 evenly through the returned random value.
48 <> The SRNG produces 96.51 bits of entropy per 32 bit number returned.
49 <> Assertion: A source of randomness with sufficient entropy passed through a cryptographic hash
50 will produce output that passes all statistical randomness tests as well as the next-bit test.
51 (state compromise)
52 <> The entropy pool is discarded after it is used once, and a new pool is
53 generated for the next random number.
54 <> ^ This means that the prior state has no bearing on the next output of the PRNG.
55 <> The SRNG destroys its own state after the random number is generated so that
56 the state used to generate it does not persist in memory.
57
58 * Due to the derivation of entropy from subtle variations in the electrical state of
59 unmapped memory, this SRNG can also be considered a _hardware-based RNG_ (HWRNG).
60 */

81/*
82 Advanced Encryption Standard (AES)
83
84 AES is a form of symmetric encryption, and also is a form of block cipher.
85 Symmetric encryption means that the same key works in both directions.
86 A block cipher is an encryption algorithm that operates on data in segments (for AES, 16 bytes),
87 moving through it one segment at a time.
88
89 Symmetric encryption is usually fast, and is generally more secure for smaller key sizes.
90 AES is one of the most secure encryption systems in use today.
91 The most secure version of the algorithm is AES-256 (uses a 256-bit key).
92 AES is one of the open-source encryption schemes believed secure enough to withstand even
93 the advent of quantum computing.
94 */

270/*
271 RSA Public Key Encryption
272
273 Public key encryption is a form of asymmetric encryption.
274 This means that a key only works in one direction, and both parties need a public key
275 and a private key. The private key is used to decrypt a message and the public key is
276 used to encrypt.
277 In RSA, the public and private keys are modular inverses of each other, such that:
278 encrypted = message ** public exponent % public key, and
279 message = encrypted ** private exponent % private modulus
280 ** means power, % means modulus
281
282 65537 (and a few other Fermat primes) are commonly used as public exponents.
283 The public key (modulus) is sent in the clear and is known to to everyone.
284 The cryptographic strength of RSA comes from the difficulty of factoring the modulus.
285 Asymmetric encryption is generally VERY slow. Using even RSA-1024 on the TI-84+ CE will
286 take several seconds. For this reason, you usually do not use RSA for sustained encrypted
287 communication. Use RSA to share a symmetric key, and then use AES for future messages.
288
289 */

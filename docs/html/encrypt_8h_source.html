19/*
20 Cryptographically-Secure Random Number Generator (CSRNG)
21
22 Many of the psuedorandom number generators (PRNGs) you find in computers and
23 even the one within the C toolchain for the CE are insecure for cryptographic
24 purposes. They produce statistical randomness, but the state is generally seeded
25 using a value such as rtc_Time(). If an adversary reconstructs the seed, every
26 output of the PRNG becomes computable with little effort. These types of PRNGs
27 are called deterministic algorithms--given the input, the output is predictable.
28 These PRNGs work for games and other applications where the illusion of randomness
29 is sufficient, but they are not safe for cryptography.
30
31 A secure PRNG is an random number generator that is not only statistically random, but
32 also passes the next-bit and state compromise tests. The _next-bit test_ is defined like so:
33 given the prior output of the PRNG (bits 0=>i), the next bit (i+1) of the output cannot be
34 predicted by a polynomial-time statistical test with a probability non-negligibly greater than 50%.
35 In simpler terms, a secure PRNG must be unpredictable, or "entropic".
36 The _state compromise test_ means that an adversary gaining knowledge of the initial state of
37 the PRNG does not gain any information about its output.
38
39 The SRNG previded by HASHLIB solves both tests like so:
40 (next-bit)
41 <> The SRNG's output is derived from a 119-byte entropy pool created by reading
42 data from the most entropic byte located within floating memory on the device.
43 <> The "entropic byte" is a byte containing a bit that that, out of 1024 test reads,
44 has the closest to a 50/50 split between 1's and 0's.
45 <> The byte containing that bit is read in xor mode seven times per byte to offset
46 any hardware-based correlation between subsequent reads from the entropic byte.
47 <> The entropy pool is then run through a cryptographic hash to spread that entropy
48 evenly through the returned random value.
49 <> The SRNG produces 96.51 bits of entropy per 32 bit number returned.
50 <> Assertion: A source of randomness with sufficient entropy passed through a cryptographic hash
51 will produce output that passes all statistical randomness tests as well as the next-bit test.
52 (state compromise)
53 <> The entropy pool is discarded after it is used once, and a new pool is
54 generated for the next random number.
55 <> ^ This means that the prior state has no bearing on the next output of the PRNG.
56 <> The SRNG destroys its own state after the random number is generated so that
57 the state used to generate it does not persist in memory.
58
59 * Due to the derivation of entropy from subtle variations in the electrical state of
60 unmapped memory, this SRNG can also be considered a _hardware-based RNG_ (HWRNG).
61 */

82/*
83 Advanced Encryption Standard (AES)
84
85 AES is a form of symmetric encryption, and also is a form of block cipher.
86 Symmetric encryption means that the same key works in both directions.
87 A block cipher is an encryption algorithm that operates on data in segments (for AES, 16 bytes),
88 moving through it one segment at a time.
89
90 Symmetric encryption is usually fast, and is generally more secure for smaller key sizes.
91 AES is one of the most secure encryption systems in use today.
92 The most secure version of the algorithm is AES-256 (uses a 256-bit key).
93 AES is one of the open-source encryption schemes believed secure enough to withstand even
94 the advent of quantum computing.
95 */
96

271/*
272 RSA Public Key Encryption
273
274 Public key encryption is a form of asymmetric encryption.
275 This means that a key only works in one direction, and both parties need a public key
276 and a private key. The private key is used to decrypt a message and the public key is
277 used to encrypt.
278 In RSA, the public and private keys are modular inverses of each other, such that:
279 encrypted = message ** public exponent % public key, and
280 message = encrypted ** private exponent % private modulus
281 ** means power, % means modulus
282
283 65537 (and a few other Fermat primes) are commonly used as public exponents.
284 The public key (modulus) is sent in the clear and is known to to everyone.
285 The cryptographic strength of RSA comes from the difficulty of factoring the modulus.
286 Asymmetric encryption is generally VERY slow. Using even RSA-1024 on the TI-84+ CE will
287 take several seconds. For this reason, you usually do not use RSA for sustained encrypted
288 communication. Use RSA to share a symmetric key, and then use AES for future messages.
289
290 */
291
